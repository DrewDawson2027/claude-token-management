# Source Inventory

This inventory covers every `.py` and `.js` file under `src/`. Each entry compresses the required analysis dimensions into one line: purpose, lifecycle, main inputs and outputs, state, dependencies, failure mode, and the biggest quality signal.

## Guards

- `src/hooks/guards/token-guard.py` (`PreToolUse: Task`): the real agent-spawn enforcement center; reads hook stdin plus `token-guard-config.json` and session-state JSON, writes allow/block audit and Explore-target metadata, depends on `guard_contracts`, `guard_events`, `guard_normalize`, and `hook_utils`, and is large but internally structured; failures defer to configurable shadow/fail-open behavior (`src/hooks/guards/token-guard.py:2-39`, `src/hooks/guards/token-guard.py:431-534`, `src/hooks/guards/token-guard.py:665-1224`).
- `src/hooks/guards/budget-guard.py` (claims `PreToolUse: .*`): reads stdin, `budgets.json`, `token-guard-config.json`, cost cache, and session JSONL, may shell out to `cost_runtime.py`, emits warn/block JSON and optional hot-path blocking, but is not wired in the snapshot and imports `record_hook_outcome` from `hook_utils.py`, which does not define it (`src/hooks/guards/budget-guard.py:3-24`, `src/hooks/guards/budget-guard.py:61-107`, `src/hooks/guards/budget-guard.py:161-242`, `src/hooks/guards/budget-guard.py:319-413`, `src/hooks/infrastructure/hook_utils.py:40-165`).
- `src/hooks/guards/read-efficiency-guard.py` (`PreToolUse: Read`): blocks duplicate reads and long sequential-read bursts, reads per-session read state plus token-guard Explore metadata, writes back read counters, and is one of the few live hard anti-waste guards; code is straightforward and portable (`src/hooks/guards/read-efficiency-guard.py:3-23`, `src/hooks/guards/read-efficiency-guard.py:90-119`, `src/hooks/guards/read-efficiency-guard.py:147-246`, `config/settings-snapshot.json:243-249`).
- `src/hooks/guards/credential-guard.py` (`PreToolUse: Write/Edit/MultiEdit/Bash`): regex-based secret leak prevention for file edits and commits; reads stdin or staged diffs, emits block reasons on stdout, keeps no persistent state, and is small and testable but heuristic-heavy (`src/hooks/guards/credential-guard.py:3-9`, `src/hooks/guards/credential-guard.py:11-24`).
- `src/hooks/guards/risky-command-guard.py` (`PreToolUse: Bash`): pattern-matches catastrophic or risky commands, blocks the worst set and warns on a second tier, has no persistent state, and is easy to reason about; precision depends entirely on regex coverage (`src/hooks/guards/risky-command-guard.py:2-12`, `src/hooks/guards/risky-command-guard.py:19-24`).
- `src/hooks/guards/circuit_breaker.py` (library/helper): shared bypass logic that suppresses repeatedly failing hooks after three failures in five minutes; reads and writes a local JSON state file and is useful operationally but only if other hooks call it consistently (`src/hooks/guards/circuit_breaker.py:1-20`).
- `src/hooks/guards/review-gate.py` (claims `PreToolUse: Write/Edit/MultiEdit`): blocks file modifications while a `review-pending` flag exists; reads a flag file and expires it after 15 minutes, but is not wired in the snapshot so the enforcement story is incomplete (`src/hooks/guards/review-gate.py:2-13`, `src/hooks/guards/review-gate.py:21-67`, `config/settings-snapshot.json:185-249`).

## Tracking

- `src/hooks/tracking/session-tracker.py` (`SubagentStop`): incremental per-session token tracker that tails transcript JSONL by file offset, updates a hot session JSON file, and records completion counts; strong practical design, but it also imports missing `record_hook_outcome` on the way out (`src/hooks/tracking/session-tracker.py:2-9`, `src/hooks/tracking/session-tracker.py:72-249`, `src/hooks/tracking/session-tracker.py:255-269`).
- `src/hooks/tracking/agent-metrics.py` (`SubagentStop`): parses actual transcript usage into `agent-metrics.jsonl`; inputs are transcript JSONL and pricing constants, outputs are durable per-agent usage records, and quality is good because it measures from recorded facts rather than estimates (`src/hooks/tracking/agent-metrics.py:2-8`, `src/hooks/tracking/agent-metrics.py:16-24`, `src/hooks/tracking/agent-metrics.py:138-205`).
- `src/hooks/tracking/session-summary.py` (`Stop`): finalizes day-level session analytics, writes one JSONL summary row, and deletes the hot-layer state file; simple, focused, and appropriately fail-open (`src/hooks/tracking/session-summary.py:2-10`, `src/hooks/tracking/session-summary.py:30-102`).
- `src/hooks/tracking/cost-tagger.py` (`SessionStart`): infers a cost center from `PWD`, writes `{session_id}-cost-tag.json`, and is useful for attribution; docstring claims `budget-guard` consumes these tags, but the budget guard code does not (`src/hooks/tracking/cost-tagger.py:2-18`, `src/hooks/tracking/cost-tagger.py:90-123`, `src/hooks/guards/budget-guard.py:61-161`).
- `src/hooks/tracking/token-analytics.py` (CLI, no hook): command surface over accumulated analytics for status, week, range, savings, and export workflows; useful as an operator tool, not part of live enforcement (`src/hooks/tracking/token-analytics.py:2-24`).
- `src/hooks/tracking/session-slo-check.py` (claims `SessionStart`): runs `observability.py` in fast mode and warns on red SLOs; concise and sensible, but dormant in the snapshot (`src/hooks/tracking/session-slo-check.py:2-16`, `config/settings-snapshot.json:300-325`).

## Routing

- `src/hooks/routing/model-router.py` (`PreToolUse: Task`): enforces a tiny routing contract, effectively "only sonnet or haiku" plus a hard `Explore -> haiku` rule; reads stdin and local settings JSON, emits block JSON, and is clear but far simpler than the surrounding policy story suggests (`src/hooks/routing/model-router.py:28-83`).
- `src/hooks/routing/routing-reminder.py` (claims `UserPromptSubmit`): injects a routing reminder every prompt and a larger refresh every 25 messages; it writes only to stdout and counter files, but the copied snapshot does not have a matching event registration (`src/hooks/routing/routing-reminder.py:2-24`, `config/settings-snapshot.json:1-325`).

## Ops

- `src/hooks/ops/ops_sources.py` (CLI/library): shared adapter layer for cost files, hook state, inbox paths, subprocess reads, and source freshness; high leverage and reusable, with direct coupling back into `~/.claude` paths (`src/hooks/ops/ops_sources.py:1-24`, `src/hooks/ops/ops_sources.py:132-206`).
- `src/hooks/ops/ops_alerts.py` (CLI/library): evaluates budget, burn-rate, anomaly, data-quality, and fault alerts; writes `alerts.jsonl` and `alert-state.json`, optionally delivers to stderr or inbox, and is operationally solid but not snapshot-wired (`src/hooks/ops/ops_alerts.py:2-31`, `src/hooks/ops/ops_alerts.py:92-156`, `src/hooks/ops/ops_alerts.py:194-291`).
- `src/hooks/ops/ops_aggregator.py` (CLI/library): builds a cached operational snapshot using parallel cost queries and local log scans; outputs a rich unified view, but its value depends on the freshness of many file-backed sources (`src/hooks/ops/ops_aggregator.py:1-33`, `src/hooks/ops/ops_aggregator.py:124-273`).
- `src/hooks/ops/ops_trends.py` (CLI/library): scans project JSONL files into daily rolling token and cost series; writes no persistent state directly except via cache callers, and the logic is practical but locked to local file layout and simplified cost formulas (`src/hooks/ops/ops_trends.py:2-19`, `src/hooks/ops/ops_trends.py:75-157`, `src/hooks/ops/ops_trends.py:206-236`).
- `src/hooks/ops/ops_recap.py` (CLI/library): reconstructs a session narrative by reading audit, metrics, self-heal, and alert logs; good for retrospectives and debugging, but only as good as the completeness of the underlying logs (`src/hooks/ops/ops_recap.py:1-24`, `src/hooks/ops/ops_recap.py:60-164`).

## Infrastructure

- `src/hooks/infrastructure/guard_contracts.py` (library): schema and normalization helpers for guard records; this is one of the better architectural choices because it centralizes log shape normalization (`src/hooks/infrastructure/guard_contracts.py:1-19`).
- `src/hooks/infrastructure/guard_events.py` (library): tiny JSONL append wrapper around `locked_append`; useful but minimal (`src/hooks/infrastructure/guard_events.py:1-14`).
- `src/hooks/infrastructure/guard_normalize.py` (library): shared session ID, path, and text normalization utilities; low risk and high leverage (`src/hooks/infrastructure/guard_normalize.py:1-24`).
- `src/hooks/infrastructure/hook_utils.py` (library): portable locking, atomic JSON writes, fault-tolerant JSONL reads, and default guard config; important foundation, but multiple other hooks expect helper functions it does not provide (`src/hooks/infrastructure/hook_utils.py:1-9`, `src/hooks/infrastructure/hook_utils.py:40-165`).
- `src/hooks/infrastructure/hook_audit.py` (library): shared standardized audit writer for hook decisions; good schema intent, but value depends on consistent adoption across hooks (`src/hooks/infrastructure/hook_audit.py:2-20`).
- `src/hooks/infrastructure/hook_health.py` (CLI/library): aggregates hook counters and audit latency into an SLO summary; useful operator tooling, not part of the live session path (`src/hooks/infrastructure/hook_health.py:2-14`).
- `src/hooks/infrastructure/hook-doctor.py` (CLI): diagnostic sweep over file existence, permissions, dependencies, state, lock files, and disk usage; high signal operational script, but separate from automated health enforcement (`src/hooks/infrastructure/hook-doctor.py:2-18`).
- `src/hooks/infrastructure/read-cache.py` (claims `PreToolUse: Read`): advisory semantic read cache keyed by `(path, mtime, size)`; writes cache entries under `~/.claude/cache/read-results`, but is not wired and imports a missing helper on exit (`src/hooks/infrastructure/read-cache.py:2-16`, `src/hooks/infrastructure/read-cache.py:176-187`, `config/settings-snapshot.json:243-249`).
- `src/hooks/infrastructure/result-compressor.py` (claims `PostToolUse`): context-bloat detector for oversized tool results; prints warnings and intends to track context growth, but is not wired and would fail if activated because `hook_utils.py` lacks both `track_context_growth` and `record_hook_outcome` (`src/hooks/infrastructure/result-compressor.py:2-13`, `src/hooks/infrastructure/result-compressor.py:56-60`, `src/hooks/infrastructure/result-compressor.py:95-108`, `src/hooks/infrastructure/hook_utils.py:40-165`).
- `src/hooks/infrastructure/auto-review-dispatch.py` (`PostToolUse: Bash`): writes mandatory review actions after commits and PR commands; good producer logic, but extracted project lacks the consumer half of the queue flow (`src/hooks/infrastructure/auto-review-dispatch.py:2-10`, `src/hooks/infrastructure/auto-review-dispatch.py:22-47`).
- `src/hooks/infrastructure/build-chain-dispatcher.py` (`SubagentStop`): writes chained post-build actions to the same mandatory queue and persists chain state; same producer-only limitation as above (`src/hooks/infrastructure/build-chain-dispatcher.py:2-16`, `src/hooks/infrastructure/build-chain-dispatcher.py:67-83`, `src/hooks/infrastructure/build-chain-dispatcher.py:153-188`).
- `src/hooks/infrastructure/chain-advance.py` (helper): advances a queue-chain state file when a consumer reports completion; simple and useful, but not enough to make the queue whole on its own (`src/hooks/infrastructure/chain-advance.py:2-9`, `src/hooks/infrastructure/chain-advance.py:17-46`).
- `src/hooks/infrastructure/pre-compact-context.py` (helper invoked from shell): snapshots recent context before compaction into markdown files; narrowly scoped, fail-open, and dependent on surrounding shell glue (`src/hooks/infrastructure/pre-compact-context.py:3-10`).
- `src/hooks/infrastructure/self-heal.py` (claims `SessionStart`): validates files, smoke-tests hooks, cleans stale state, repairs permissions, and logs results; ambitious and substantial, but currently dormant in the snapshot (`src/hooks/infrastructure/self-heal.py:3-13`, `config/settings-snapshot.json:300-325`).

## Core Runtime

- `src/scripts/core/cost_runtime.py` (CLI/runtime): the central cost engine that owns local usage parsing, ccusage integration, budget status, caching, statusline generation, burn/anomaly checks, and proactive alert shell-outs; very high value, but still monolithic and partly duplicative of `cost_data.py` (`src/scripts/core/cost_runtime.py:64-67`, `src/scripts/core/cost_runtime.py:205-291`, `src/scripts/core/cost_runtime.py:608-783`, `src/scripts/core/cost_runtime.py:917-974`, `src/scripts/core/cost_runtime.py:1717-1752`).
- `src/scripts/core/pricing.py` (library): normalizes model names and computes equivalent USD cost from token usage; cleanest module in the project (`src/scripts/core/pricing.py:2-13`, `src/scripts/core/pricing.py:94-211`).
- `src/scripts/core/cost_base.py` (library): extracted shared primitives for JSON, timestamps, and IDs; a reasonable refactor seam (`src/scripts/core/cost_base.py:1-12`).
- `src/scripts/core/cost_data.py` (library): intended extracted data I/O layer for `cost_runtime.py`; contains its own `UsageRecord`, parsing, and `run_ccusage`, but the main runtime never adopted the extraction, so this reads as a partial refactor (`src/scripts/core/cost_data.py:1-13`, `src/scripts/core/cost_data.py:46-333`, `src/scripts/core/cost_runtime.py:173-343`, `src/scripts/core/cost_runtime.py:544-652`).
- `src/scripts/core/cost_doctor.py` (CLI): smoke-check script for `cost_runtime.py`, `ccusage`, and basic environment presence; useful, but minimal (`src/scripts/core/cost_doctor.py:1-24`).
- `src/scripts/core/batch_queue.py` (CLI): queue for deferred low-priority work aimed at Anthropic's Batch API discount path; conceptually interesting, but no live integration path from the hook system into it (`src/scripts/core/batch_queue.py:2-13`, `src/scripts/core/batch_queue.py:21-24`).
- `src/scripts/core/policy_engine.py` (CLI/library): governance and redaction tooling around policies, signed artifacts, and reports; adjacent to cost management but not deeply integrated into the live guard path (`src/scripts/core/policy_engine.py:2-24`).
- `src/scripts/core/observability.py` (CLI/library): SLO, parity, audit, and timeline reporting across the broader `.claude` environment; useful operator tooling and a dependency of `session-slo-check.py`, but outside the core token-enforcement loop (`src/scripts/core/observability.py:2-24`).

## Analytics

- `src/scripts/analytics/savings_calculator.py` (CLI): estimates savings by correlating guard block events with observed agent cost medians; sensible and grounded in actual metrics, but still a heuristic counterfactual model (`src/scripts/analytics/savings_calculator.py:2-9`, `src/scripts/analytics/savings_calculator.py:20-24`).
- `src/scripts/analytics/calculate_real_savings.py` (CLI): one-off corrective script for a better savings methodology; useful as forensic tooling, not a maintained subsystem (`src/scripts/analytics/calculate_real_savings.py:2-15`).
- `src/scripts/analytics/token_snapshots.py` (CLI): generates daily, weekly, and monthly snapshot files from session JSONL; strong historical spine for retrospectives (`src/scripts/analytics/token_snapshots.py:2-14`).
- `src/scripts/analytics/subagent-usage.py` (CLI): summarizes `agent-metrics.jsonl` by subagent type, cost, and frequency; compact and practical (`src/scripts/analytics/subagent-usage.py:2-17`).

## Inventory Conclusion

The source inventory shows a system with a credible operational spine and a wider ring of partially integrated experiments. The live value comes from a subset of the tree, not the entire extracted directory.
